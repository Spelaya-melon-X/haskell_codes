**`newtype`**:
- Один конструктор, одно поле
- Нулевая стоимость в runtime (исчезает после компиляции)
- Используется для обёрток и безопасности типов

**`data`**:
- Может иметь несколько конструкторов и полей
- Имеет overhead в runtime
- Используется для реальных структур данных

**Пример:**
```haskell
newtype Age = Age Int           -- Просто обёртка для Int

-- c++ аналогия : 
using Age = int ; 


data Person = Person String Age -- Настоящая структура данных

-- c++ аналогия : 
struct Person {
    std::string name ; 
    Age age ; 
}; 

```




## **Про синтаксис `Age = Age Int`:**
- **Первый `Age`** — имя типа
- **Второй `Age`** — имя конструктора  
- **`Int`** — тип поля

```haskell
-- Как будто объявляем:
struct Age {  -- Тип
    Age(int value);  // Конструктор
};
```

## **let**
- Выражение `let` в `Haskell` используется для связывания локальных переменных внутри выражения. 
Оно позволяет определять имена для значений или функций, которые находятся только в области видимости внутри части выражения let.

- Внутри блоков do оператор `let` используется без явного ключевого слова `in`. Привязки, вводимые `let`, распространяются на все последующие операторы внутри этого блока `do`.
